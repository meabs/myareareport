# Submission Checklist

## Before Submitting

### Legal and Compliance
- [ ] Privacy policy page live and linked
- [ ] Terms of use page live and linked
- [ ] Data sources page live and linked
- [ ] No unsupported claims in descriptions or UI
- [ ] No "safe/unsafe/dangerous" language in UI

### App Description
- [ ] Short description written (see app-metadata-draft.md)
- [ ] Long description written
- [ ] Required caveat text included

### Screenshots
- [ ] All screenshots from screenshot-checklist.md captured
- [ ] Screenshots show real data, not placeholder content

### Functional Checks
- [ ] Postcode search works
- [ ] Invalid postcode shows friendly error
- [ ] Report page renders all sections
- [ ] Sources and caveats visible throughout
- [ ] Mobile viewport works

### Data and Privacy
- [ ] No user data stored permanently
- [ ] No PII logged to Sentry
- [ ] No tracking cookies

### Infrastructure
- [ ] Deployment checklist completed
- [ ] Monitoring in place (Sentry)
- [ ] Redis caching working
